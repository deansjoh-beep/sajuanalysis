# Release Notes — 2026-04-06 (2)

## 프리미엄 리포트 대운 연도 시스템 만세력 기준으로 고정

### 문제
프리미엄 리포트의 대운 연도(예: 경오대운 2017~2026)가 시스템 만세력(경오대운 2016~2025)과 1년 차이 발생.

**원인**:
1. `daeunContext` 문자열에 캘린더 연도(`startYear`)가 없고 나이(`startAge`)만 포함 → AI가 생년에 나이를 더해 연도를 스스로 계산
2. `d.stem.hangul` / `d.branch.hangul` 프로퍼티 접근이 잘못됨 (`getDaeunData`는 plain 한자 string 반환) → 한글 변환이 `undefined`로 깨짐

---

### 변경 파일

| 파일 | 내용 |
|---|---|
| `src/lib/generatePremiumReport.ts` | `daeunContext` / `daeunText`에 `(startYear~startYear+9년)` 포함, `hanjaToHangul` 변환 수정 |
| `src/lib/promptBuilders.ts` | 시스템 지시에 대운 연도 고정 규칙 추가, `[DAEUN_START]` 포맷에 연도 범위 명시 |

---

### 수정 상세

#### `generatePremiumReport.ts`
```
// 이전
`${d.startAge}세 대운: ${d.stem.hangul}(${d.stem.hanja})${d.branch.hangul}(${d.branch.hanja})`

// 이후
const stemHangul = hanjaToHangul[d.stem] || d.stem;
const branchHangul = hanjaToHangul[d.branch] || d.branch;
`${d.startAge}세(${d.startYear}~${d.startYear + 9}년) 대운: ${stemHangul}(${d.stem})${branchHangul}(${d.branch})`
```

#### `promptBuilders.ts` — 신규 규칙
```
대운 캘린더 연도 고정 규칙(최우선):
대운 시작/종료 연도는 반드시 [대운 흐름] 데이터에 제공된 "(YYYY~YYYY년)" 값을 그대로 사용하세요.
절대로 생년에 나이를 더하여 연도를 자체 계산하지 마세요.
```

### 효과
- AI가 시스템 만세력 엔진(`getDaeunData`)이 계산한 연도를 그대로 사용
- 대운 연도 불일치 해소
