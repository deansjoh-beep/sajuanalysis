## 확정 (Confirmed Decisions)

### 2026-07-03
- **D-0-1** 프레임워크 → **Vite SPA + Express 유지**. Next.js 이관 비용 대비 이득 없음. 결제·리포트 신규 라우팅은 Vercel Functions(`api/*`)로 붙인다.
- **D-0-2** DB → **Vercel Postgres(Neon) 이관**. Phase 2에서 `codes`+`orders`+`reports` 3테이블 스키마 + `expires_at` 자동 파기(Vercel Cron)를 관계형에서 구현. Firestore는 블로그·관리자 기존 데이터에 한해 유지, 신규 사주 데이터는 Postgres.
- **D-0-3** LLM → **Anthropic Claude 이관**. 리포트 생성·규칙 엔진 결합은 Claude로. Gemini는 실시간 상담(챗)만 유지 가능성 열어둠 — D-2-1 결정 시 최종 확정.
- **D-1-1** 야자시/조자시 → **야자시 유지** (23:00~23:59 = 다음 날 자시). 현행 `setDayZero(2)` 정책 유지. `lib/manseryeok/policy.ts`에 명문화.
- **D-1-2** 진태양시 → **서울 경도 126.98° 기본값 자동 적용**. 사용자가 다른 출생지를 입력하는 UI는 이후 옵션으로 노출.
- **D-1-3** 서머타임/표준시 → **도입**. `getAdjustedTime()` orphan 상태를 해소하고 실제 `getSajuData`에 결합. 정본은 KASI 문헌. 조건식 상한값(1961-08-10 이후 무한 -30분) 정정 필요.
- **D-1-4** 검증 데이터셋 → **A안 KASI Open API**. 절기 시각을 API로 수집해 자체 절기 테이블 구축 후 lunar-javascript 결과 대조. ✅ 2026-07-04 KASI 키 `.env.local`에 등록됨.

### 2026-07-06
- **D-1-6** 격국·용신 정식 판정 유파 → **(a) 자평명리 표준** 확정. Phase 3-1 규칙 엔진과 「명리 판단 기준서」는 자평 표준을 기본 축으로 하며, 조후는 자평 체계 안의 보조 판단으로 위치시킨다(우선순위 세부는 기준서 4·6장에서 문답으로 확정). 기준서 기획: `docs/myeongri-standard-plan.md`. 세부 판정 지침은 동 문서 "확정 지침(문답 기록)" 섹션에 누적한다.

### 2026-07-04
- **격국·용신 v1 = provisional 휴리스틱 확정 (기술 판단, 플랜 1-2 준수)** — 만세력과 달리 격국·용신은 검증된 단일 정답이 없고 유파(자평·궁통보감·적천수 등)마다 판정이 근본적으로 다르다. 따라서 v1은 기존 saju.ts 휴리스틱을 `src/lib/analysis/gyeokyongshin.ts`로 정리·구조화하되 **모든 출력에 `provisional: true`·유파 경고문**을 부착하고, 리포트 프롬프트는 "참고 경향"으로만 사용한다(단정 금지). 정식 규칙 엔진은 Phase 3.
  - 추가된 것: 타입 정의, 신강/신약 투명화(득령·득지·득세 3분해), 격명 정본화(월지 본기 비견=건록격·겁재=양인격), 레거시 어댑터(기존 App.tsx·리포트·택일 무변경). 수치·최종 선택 로직은 종전과 동일(런타임 통변 텍스트 불변).
- **D-1-6 (신규 대기) 격국·용신 정식 판정 유파** — Phase 3 규칙 엔진이 채택할 유파를 OWNER가 결정해야 한다(자평명리 표준 / 궁통보감 조후 중심 / 혼합). 결정 전까지 v1 provisional 유지. **Phase 1~2 비블로킹**.
- **D-1-4 보강 (KASI 공백구간 검증 방침, 기술 판단)** — KASI 실측 커버리지는 2000~2028년뿐. 공백구간(1900~1999, 2029~2030)은 다음과 같이 확정한다:
  - **1차 절기 기준은 전 구간에서 lunar-javascript 단일 유지** (KASI 창 실측: 중앙값 0.27분, 절 348건 99.7%가 ≤1분).
  - **공백구간은 독립 천문계산(`jieqi-astro.ts`, Meeus ±14분)으로 전수 감시** — `scripts/kasi/watch-gap-years.ts`가 절(節) 1,224건을 대조. 실측 결과 |오차| 평균 3.73분·최대 13.92분(천문계산 자체 정밀도 이내), 30분 임계 초과 0건 → lunar-javascript에 KASI형 오류(수십 분~수 시간) 징후 없음.
  - 결론: 공백구간에 별도 데이터 발주 불필요. 감시 스크립트를 회귀 체크에 포함.
- **D-1-1 이행 명세 정정** — 종전 코드의 `setDayZero(2)` 호출은 lunar-javascript에 존재하지 않는 메서드로 no-op이었고, 동작은 라이브러리 기본값에 우연히 의존하고 있었다. 실존 API `EightChar.setSect(2)`(유파 2: 시주는 익일 子時 간, 일주는 당일 유지 = 야자시)로 명시 고정. 동작 변화 없음(기본값과 동일).
- **D-0-4** `src/App.tsx` 3,895줄 **분해 착수 승인**. Phase 1 리팩터링 첫 단계로 도메인별(챗/리포트/블로그/관리자/사주계산 UI) 파일 분리. LLM 호출 로직은 서버 프록시로 이관.
- **D-0-5** `server.ts` ↔ `api/*` 이중 구현 → **`api/*` (Vercel Functions) 단일화**. `server.ts`는 로컬 dev용 Vite 미들웨어 역할만 유지, 자체 라우트는 `api/*.ts`로 이관하고 dev 환경에서도 동일 코드가 실행되도록 재구성.
